# Тестовое Задание - Python Backend

**Срок сдачи:** 3 дня  
**Стек:** Python >=3.11, FastAPI, Docker

---
## Контекст

Ты делаешь внутренний сервис для команды разработки. Команда создаёт задачи вручную, назначает их на себя и двигает по статусам в процессе работы. Ничего лишнего - только то, что нужно чтобы не потерять что делается и в каком состоянии. 

---

## Что Нужно Реализовать

Сервис должен позволять:

- Создать задачу с названием, описанием и исполнителем
- Получить список задач с фильтрацией по статусу и исполнителю
- Получить одну задачу по идентификатору
- Изменить статус задачи
- Удалить задачу 

Статусы:  `todo` -> `in_progress` -> `review` -> `done`. Переход назад из `done` невозможен - возвращай понятную ошибку.

Поля модели можно расширять, если считаешь, что какое-то поле сделает сервис лучше, добавляй и объясняй в README почему.

---

## Требования

### Обязательно
- Стек: Python + FastAPI. Всё остальное - на твоё усмотрение
- Проект поднимается одной командой `docker compose`
- Валидация входных данных и понятные сообщения об ошибках
- README: как запустить, какие решения принял и почему

### Бонус
- Кэширование списка задач в Redis - с обоснованием TTL в README
- Тесты на создание задачи и смену статуса
- CI-пайплайн (GitHub Actions или GitLab CI) - достаточно запуска линтера или тестов
